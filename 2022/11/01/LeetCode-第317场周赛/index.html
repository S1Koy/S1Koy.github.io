LeetCode 第317场周赛

发表于 2022-11-01 分类于 LeetCode 阅读次数： Valine：

A

思路：模拟题，第一眼还想着求数位之和是不是$3$的倍数，然后发现傻了。又能被$3$整除，又能被$2$整除，即被$LCM(2,3)$最小公倍数整数。这里有个除以$0$的操作，坑。

B

思路：感觉写着挺麻烦，先统计流行度最高的创作者有哪些，然后再统计他们视频播放量最高对应的$id$。用哈希表搞一下，这里一开始存创作者的时候用的列表，但是播放量并不是都是增加的，存在$0$值，导致可能出现重复，所以要用集合。

C

思路：刚打完校赛感觉脑子没了，看了室友的题，没怎么思考。其实就是不断的进位，因为后续的数字之和只会更大，只能靠进位来变小。

D

思路：一开始的思路是在叶子节点存路径上都有哪些点，然后逐个遍历再计算，复杂度不太行。

看了题解：两种做法。

解法一：

因为每次只删除一个节点，且不可能是根节点，因此，删除一个点后的只能是从左边或者右边走的最大深度，所以我们可以进行两次遍历：第一次遍历，先左后右；第二次遍历，先右后左。每次遍历的时候，每个节点更新为当前遍历过的的最大深度，分别用两个数组保存。然后询问的时候，肯定是两个答案中最大的那一个。

解法二：

$DFS$序，对于一个子树来说，这个树的$DFS$序一定是连续的，也就是给定一个区间，删除一段连续的区间，求剩余区间的最大值。

如果所示：将红框子树删除，也就是$DFS$序从$[4,6]$区间被删除，答案只能是$[1,3]$和$[7,8]$对应的最大值，因此我们用$DFS$序存相应的深度，然后存某个节点的开始时间序$st$和结束时间序$ed$，然后答案就是$[1,st-1]$和$[ed+1,t]$的最大值，区间最大值可以提前处理出来。
